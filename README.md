# Estudo sobre fraudes em cartões bancarios.

Este projeto simula o combate a fraudes financeiras, focando na identificação de comportamentos anômalos e na redução de falsos positivos através de análise de dados. A correlação de eventos é aprimorada ao cruzar indicadores dispersos e estabelecer linhas de base estatísticas, permitindo a detecção automática de ameaças

## 🛠️ Tecnologias e Ferramentas

O projeto foi desenvolvido em ambiente em nuvem utilizando as seguintes tecnologias e bibliotecas:

* **Ambiente de Desenvolvimento:** [Google Colab](https://google.com)
* **Linguagem:** [Python](https://python.org)
* **Manipulação e Análise de Dados:** [Pandas](https://pydata.org) | [NumPy](https://numpy.org)
* **Visualização de Dados:** [Matplotlib](https://matplotlib.org)
* **Machine Learning:** [Scikit-Learn (sklearn)](https://scikit-learn.org)
* **Armazenamento:** Arquivos em formato [CSV](https://wikipedia.org)

## 📋 Etapas do Desenvolvimento

O ciclo de vida do projeto foi dividido em quatro fases estruturadas:

1. **Coleta de Dados:** Download da base de dados oficial de fraudes no [Kaggle](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud).
2. **Ingestão e Engenharia de Dados:** Carregamento do arquivo e transformações necessárias para modelagem.
3. **Análise Exploratória (EDA):** Análise estatística e visual das variáveis para entender o comportamento das fraudes.
4. **Modelagem e Validação:** Testes de algoritmos de Machine Learning e avaliação de métricas de desempenho.

> 💡 **Nota de Execução:** Os detalhes técnicos, códigos e procedimentos detalhados de cada etapa estão documentados diretamente no arquivo do notebook deste repositório.
